# Product Requirements Document (PRD) - Multi-Page Version
**Project Name:** Battery Production Management System (BPMS) v2.0 (MPA)
**Document Version:** 2.0

## 1. Product Overview
BPMS v2.0 adalah sistem manajemen produksi baterai berbasis **Multi-Page Application (MPA)**. Berbeda dengan versi sebelumnya, setiap modul utama kini memiliki file fisik sendiri untuk meningkatkan isolasi kode dan kemudahan pemeliharaan, namun tetap tersinkronisasi menggunakan satu basis data JSON melalui API pusat.

## 2. Tech Stack & Architecture (MPA)
* **Architecture:** Multi-Page Application (MPA) dengan file HTML terpisah.
* **Frontend:** HTML5, CSS3 (Shared Global Styles), Vanilla JavaScript (Modular).
* **Routing:** Navigasi tradisional menggunakan tag anchor (`<a href="...">`) antar file.
* **Backend (API):** `state.php` (RESTful API) untuk sinkronisasi data antar halaman.
* **Data Persistence:** * *Client-Side:* LocalStorage sebagai cache transisi antar halaman.
    * *Server-Side:* `data/state.json` sebagai sumber kebenaran data tunggal (*Single Source of Truth*).

## 3. File Structure & Sitemap
Sistem akan dipecah menjadi file-file berikut:
1.  `index.html`: Dashboard & Overview (Ringkasan LVC semua LINE).
2.  `database-pcs.html`: Pengelolaan Master Data Kapasitas Mesin.
3.  `data-order.html`: Import dan manajemen data order/loading bulanan.
4.  `setting-capacity.html`: Pengaturan kalender kerja dan shift per LINE.
5.  `lvc-analysis.html`: Visualisasi detail Loading vs Capacity dan chart.
6.  `man-power.html`: Kalkulasi kebutuhan tenaga kerja.
7.  `shared-components.js`: Script bersama untuk memuat Header, Sidebar, dan sinkronisasi State API di setiap halaman.

## 4. Core Logic & Transitions

### 4.1. Shared State Management (Critical)
Karena bersifat MPA, setiap kali halaman berpindah, memori JavaScript akan terhapus.
* **Requirement:** Setiap file HTML harus menjalankan fungsi `syncState()` saat `window.onload`.
* **Logic:** Fungsi ini mengambil data terbaru dari `state.php`, menyimpannya ke `localStorage`, lalu merender komponen sesuai data tersebut.

### 4.2. Database PCS (database-pcs.html)
* **Fungsi:** CRUD (Create, Read, Update, Delete) Master PCS.
* **Calculation Logic:** Tetap mempertahankan rumus otomatis:
    * `BATT/H` = `3600 / CT`
    * `S1/S2/S3 Capacity` = `(Menit Shift * 60) / CT`
* **Save Action:** Menekan tombol "Save" akan mengirimkan seluruh objek PCS ke `state.php`.

### 4.3. Data Order & Parsing (data-order.html)
* **Item Extraction Logic:** Algoritma pemotongan string tetap sama (Digit ke-4 untuk Tech, ke-6 Separator, 8-14 Type, 16-17 Plate).
* **Navigation:** Setelah upload berhasil, pengguna diberikan opsi untuk langsung berpindah ke `lvc-analysis.html`.

### 4.4. LVC Analysis (lvc-analysis.html)
* **Logic:** Melakukan kalkulasi beban kerja dengan membandingkan data dari `data-order.html` dan `setting-capacity.html`.
* **Visualisasi:** Menggunakan modul Chart yang dimuat ulang setiap kali halaman dibuka berdasarkan data di `state.json`.

## 5. UI/UX Requirements
* **Global Sidebar:** Sidebar harus konsisten di semua file. State sidebar (collapsed/expanded) disimpan di `localStorage` agar tidak berubah saat pindah halaman.
* **Active Link:** Menu di sidebar harus mendeteksi URL saat ini untuk memberikan class `.active` pada menu yang sesuai.
* **Loading State:** Karena MPA melakukan request ke server setiap pindah halaman, harus ada indikator loading yang halus (skeleton screen atau spinner) saat data di-fetch dari `state.php`.

## 6. Backend API Requirements (state.php)
API tidak berubah secara signifikan, namun harus mendukung:
* **CORS:** Jika file diakses dari sub-domain berbeda (opsional).
* **Atomic Write:** Memastikan saat satu halaman melakukan POST, data di halaman lain tetap sinkron saat di-refresh.
